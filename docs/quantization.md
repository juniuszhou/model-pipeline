# quantization

## 不同方法比较

方法,操作方式,精度损失,实际可用性
简单舍入,直接四舍五入,极大,基本不可用
Naive Uniform,  简单均匀映射,   很大,  勉强可用（8bit）
GPTQ,   Hessian + 逐列误差补偿, 中等,   很好
AWQ,    保护重要激活权重,   较小,  目前最佳
QAT,    量化感知训练（模拟量化）,   最小,   最好，但成本高


总结一句话区别：

GPTQ 是“数学补偿派”：用二阶优化全局最小化每一层的输出误差。

AWQ 是“重要性保护派”：通过激活值感知，聪明地差异化对待权重，重点保护那 1% 最关键的权重。


1. 简单舍入为什么不行？
假设我们要把 FP16（16位浮点数）的权重直接转成 INT4（4位整数）：

FP16 的范围：大约 ±65504，精度很高。
INT4 的范围：只有 -8 到 +7（对称量化时）。

如果直接四舍五入：

很多很小的权重（比如 0.00123）会被直接变成 0。
一些重要的权重会被粗暴地“掰”到最近的整数格子上。

结果：模型输出会产生巨大误差，尤其在深度网络中，这种误差会层层累积，最终导致：

模型生成乱码（gibberish）
严重幻觉
逻辑崩溃
甚至直接无法连贯输出

2. 为什么 LLM 对量化误差特别敏感？

权重重要性差异极大
LLM 的权重中，只有一小部分权重对最终输出影响非常大（称为 Salient Weights），大部分权重影响较小。
简单舍入会 indiscriminately（不加区分地）破坏那些重要权重。
误差传播（Error Propagation）
Transformer 是层层堆叠的结构。第一层产生的微小误差，会被第二层放大，第三层再放大……到第 80 层时，误差可能已经巨大。
激活值（Activation）分布不均匀
不同输入下，同一权重的重要性完全不同。简单量化没有考虑“这个权重在实际使用中会被如何激活”。
离散化后的数值不匹配
直接舍入后，权重的数值分布会严重偏离原始统计分布，导致后续计算（矩阵乘法）结果偏差极大。

3. 现代量化方法是怎么解决这个问题的？
现代量化方法（GPTQ、AWQ、SmoothQuant 等）核心思路是“聪明地量化 + 补偿误差”，而不是简单舍入。
主要技术手段：

基于校准数据（Calibration Data）
用几百条真实文本跑一遍模型，观察每个权重在真实场景下的“重要程度”。
误差补偿（Error Compensation）
GPTQ：用 Hessian 矩阵（二阶导数）估算每个权重变化对输出的影响，然后把量化引入的误差“分摊”到还没量化的权重上。
AWQ：重点保护那些对激活值（Activation）影响最大的权重，给它们更高的精度。

分组量化（Group-wise）
把权重分成很多小组（group_size=128），每组共享一个 scale（缩放因子），而不是整个矩阵用同一个 scale。
零点（Zero Point）和非对称量化
更灵活地映射数值范围。
